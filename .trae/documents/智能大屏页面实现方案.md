## 页面目标
- 构建左侧 AI 助手区 + 决策卡片 + 语音音量指示，右侧 3D 建筑内部热点图 + 营销数据。
- 无需对接接口，使用高质量 Mock 数据与平滑动画，风格高端大气。

## 技术栈与约束
- Next.js 16（App Router）+ React 19，Tailwind v4 已启用。
- 现状无 3D/动画库：three.js、framer-motion 等未安装。
- 方案分两档：
  1) 基础版：纯 Tailwind/CSS/Canvas 实现拟态 3D（低依赖，效果较稳）。
  2) 进阶版：新增 three.js，使用真实 3D（质感更强，动画更丰富）。

## 信息架构
- 路由：`/smart-screen`（新增目录 `app/smart-screen/page.tsx`）。
- 组件划分：
  - 左侧：`AssistantPanel`（头像与状态）、`VoiceMeter`（音量柱/波形）、`DecisionCards`（卡片列表 + CTA）。
  - 右侧：`Building3D`（3D/拟态 3D 热点可视化）、`MarketingStats`（金额/人数/热门区域/活动）。
  - 通用：`StatTile`、`HotspotLegend`。
- 数据：`mock.ts`（包含楼层/区域/人流量/活动与营销指标）。

## 视觉与动效
- 风格：深色玻璃拟态 + 低饱和金属质感 + 霓虹点缀（青蓝/紫）。
- 动画：
  - 入场：面板与卡片淡入上浮，延迟错峰；
  - 悬停：卡片高斯模糊/光晕与微缩放；
  - 语音：音量柱/波形随帧循环模拟（可选真实麦克风振幅）；
  - 3D：热点脉冲/渐变光圈、楼层轻微漂移与摄像机缓动。
- 无库实现：CSS keyframes + requestAnimationFrame；进阶版增加 three.js 后支持更自然的缓动与材质。

## 右侧 3D 区实现细节
- 基础版（无依赖）：
  - CSS 透视 + 分层平面（楼层/房间用 div/Canvas 叠层）形成“拟态 3D”。
  - 热点：按人流量在对应区域绘制圆点/热力格，颜色随强度渐变，脉冲动画。
- 进阶版（three.js）：
  - `use client` Canvas 初始化 Scene/Camera/Renderer；简单盒体构建楼层与区域。
  - 热点：粒子或发光球（Shader/点精灵），OrbitControls 轻交互（可限制旋转）。
  - 性能：实例化几何、按层级批量更新；窗口自适应与销毁清理。

## 左侧 AI 助手区
- `AssistantPanel`：头像环形呼吸灯，状态“正在聆听”；切换为文字聊天（保留输入框与消息列表的 UI 占位）。
- `VoiceMeter`：5–12 根音量柱或波形，帧驱动模拟（可选启用 getUserMedia 读取真实输入）。
- `DecisionCards`：如“a区用户较少，是否下发半价优惠券”；支持“立即下发/稍后/忽略”操作；卡片入场错峰、状态切换动画。

## 营销数据区
- 指标：今日金额（¥）、来客数、热门区域 Top3、热门活动 Top3。
- 动效：数字滚动爬升、进度条/环形图轻动画、热点标签闪烁。

## 交互与响应式
- 布局：左右分栏（左 380–440px，右自适应），1920×1080 优化；
- 自适应：≥1440 保持双栏；≤1280 合并为上下布局；
- 主题：暗色主题变量（色板/阴影/玻璃态）。

## 无接口与数据策略
- 固定 Mock：按楼层/区域生成热点强度与营销指标；每 2–6 秒轻量波动以模拟实时。
- 本地状态：页面内 useState/Ref；不接触外部 API。

## 性能与可维护性
- 组件拆分清晰，`use client` 仅限需要的可视区域。
- 动画控制统一 Hook（节流/取消与清理）。
- 3D 进阶版采用几何复用与批量更新以控资源。

## 交付与验收
- 一个可访问的 `/smart-screen` 页面，包含：
  - 左侧完整助手区（语音动画、决策卡片、文字聊天切换形式）。
  - 右侧热点“3D”视觉（基础版或 three.js 进阶版）与营销数据面板。
- 无接口依赖，Mock 数据与动画运行流畅，视觉统一高端。
